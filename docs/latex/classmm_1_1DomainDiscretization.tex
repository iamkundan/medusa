class mm\+::\+Domain\+Discretization$<$ vec\+\_\+t $>$}
Class representing domain discretization along with an associated \doxylink{classmm_1_1DomainShape}{shape}. 

Domain discretization consists of points in the interior and on the boundary. Each point has an associated type, with types of internal points being positive and types of boundary points being negative. Boundary points also have an outer unit normal normal associated with them.

Domains can be \doxylink{classmm_1_1DomainDiscretization_afdf066978668c1b7cb276945e91eb158}{added} and \doxylink{classmm_1_1DomainDiscretization_a210e4bf7082864e4bf6e6f64080ab184}{subtracted}, nodes can be \doxylink{classmm_1_1DomainDiscretization_a3bb3a5e1d7ee168d9f5db5d8b3a86714}{added} and \doxylink{classmm_1_1DomainDiscretization_a138e02d051cbdf55d4196acfcd7f955a}{removed} (removal currently at O(\+N) cost). Support / stencil indices can be \doxylink{classmm_1_1DomainDiscretization_a42f6af1738aba241b489b75358168e36}{stored} and \doxylink{classmm_1_1DomainDiscretization_a37372a1dac0ca2b59303f1b8409099eb}{accessed}.

Additionally, domain interiors can be \doxylink{classmm_1_1GeneralFill}{filled}, \doxylink{classmm_1_1BasicRelax}{regularized} and used when \doxylink{classmm_1_1DomainDiscretization_a1169ecc9c72a73a85409a4e4d3b9039a}{computing shapes}.

Merges given discretization to the current discretization. 

The shape of current domain changes to become the union of {\ttfamily \doxylink{classmm_1_1DomainDiscretization_a0d65ae1a80ede327be0864878f09d7d8}{shape()}} and {\ttfamily d.\+shape()}. The discretization {\ttfamily d} is added to {\ttfamily \texorpdfstring{$\ast$}{*}this}. Types of nodes are preserved. This makes it possible to set custom types before the {\ttfamily +=} operation and refer to those types in the union as well.

Example\+: 
\begin{DoxyCodeInclude}{0}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordtype}{double}\ h\ =\ 0.1;}
\DoxyCodeLine{\ \ \ \ DomainDiscretization<Vec2d>\ rect\ =\ BoxShape<Vec2d>(\{-\/1,\ -\/1\},\ \{1,\ 1\}).discretizeWithStep(h);}
\DoxyCodeLine{\ \ \ \ rect.types()[rect.interior()]\ =\ 2;}
\DoxyCodeLine{\ \ \ \ DomainDiscretization<Vec2d>\ circ\ =\ BallShape<Vec2d>(\{0,\ 0\},\ 0.7).discretizeWithStep(h);}
\DoxyCodeLine{\ \ \ \ circ.types()[circ.boundary()]\ =\ -\/5;\ \ \ \textcolor{comment}{//\ -\/1\ to\ -\/4\ taken\ by\ rectangle\ sides}}
\DoxyCodeLine{\ \ \ \ circ\ +=\ rect;}
\DoxyCodeLine{\ \ \ \ std::cout\ <<\ (circ.types()\ ==\ 2).\mbox{\hyperlink{classmm_1_1DomainDiscretization_acf5e94e606f28bd3fd9622f90dc9a0c0}{size}}()\ <<\ std::endl;\ \ \textcolor{comment}{//\ nonzero\ number\ is\ printed}}

\end{DoxyCodeInclude}


More specifically, when performing {\ttfamily d1 += d2}, first, boundary nodes in {\ttfamily d2} that are contained in {\ttfamily d1} are removed. Then, all nodes from {\ttfamily d1} that are included in {\ttfamily d2} are removed. Additionally, nodes from {\ttfamily d1} are removed if they are too close to any of the nodes in {\ttfamily d2}. The threshold {\ttfamily dx} for point {\ttfamily p} in {\ttfamily d1} is computed by finding the closest neighbour to {\ttfamily p} among the remaining nodes of {\ttfamily d1} and then finding its closest neighbour among those same nodes. A factor of the distance between these last two nodes is taken as {\ttfamily dx}. The remaining nodes in d2 are then added to d1.

Load discretization from a HD5 file. 

This function looks for datasets named {\ttfamily pos}, {\ttfamily types}, {\ttfamily normals} and {\ttfamily bmap}. {\ttfamily pos} should be a {\ttfamily dim x N} dataset of doubles {\ttfamily types} and {\ttfamily bmap} should be a {\ttfamily N} datasets of ints {\ttfamily normals} should be a {\ttfamily dim x B} dataset of doubles, where {\ttfamily B} is the number of boundary nodes. This format is produced by the \doxylink{classmm_1_1HDF_aceed7a68e16d0299737e183d81bb5600}{HDF\+::write\+Domain} function.

Subtracts given discretized domain from {\ttfamily \texorpdfstring{$\ast$}{*}this}. 

The shape of current domain changes to become the difference of {\ttfamily \doxylink{classmm_1_1DomainDiscretization_a0d65ae1a80ede327be0864878f09d7d8}{shape()}} and {\ttfamily d.\+shape()}. The discretization nodes that are now outside of domain are removed and the appropriate boundary nodes from {\ttfamily d} are added. Types of nodes are preserved. This makes it possible to set custom types before the {\ttfamily -\/=} operation and refer to those types in the difference as well.

Example\+:

\end{DoxyCodeInclude}


More specifically, when performing {\ttfamily d1 -\/= d2}, nodes in d1 that are contained in {\ttfamily d2} or less than {\ttfamily dx} away from {\ttfamily d2} are removed. The local spacing {\ttfamily dx} for a point {\ttfamily p} in {\ttfamily d1} is computed by first finding the nearest boundary node in {\ttfamily d2} and then taking a fraction of the distance to its nearest boundary node. Boundary nodes from {\ttfamily d2} that are contained in {\ttfamily d1} are also added, with inverted normals.
