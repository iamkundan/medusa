class mm\+::\+Uniform\+Shape\+Storage$<$ vec\+\_\+t, Op\+Families $>$}
Efficiently stores shape functions of uniform length. 

This class is used to store shape functions (stencil weights) generated for discretizations where supports of all nodes have equal size e.\+g. the support consists of 9 closest nodes. This class is more efficient than storing the shapes in a nested type, such as {\ttfamily std\+::vector\texorpdfstring{$<$}{<}std\+::vector\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}\texorpdfstring{$>$}{>}}, see \href{http://e6.ijs.si/medusa/wiki/index.php/File:Tech_report.pdf}{\texttt{ technical report}}.

If supports of node vary in size, use \doxylink{classmm_1_1RaggedShapeStorage}{Ragged\+Shape\+Storage} instead.

{\em vec\+\_\+t} & Vector type used in computations, specifies the dimensionality of the domain and scalar type for numerical computations. \\
\hline
{\em Op\+Families} & A list of operator families for which the shapes will be stored. The basic operator families are \doxylink{structmm_1_1Lap}{Lap}, \doxylink{structmm_1_1Der1s}{Der1s} and \doxylink{structmm_1_1Der2s}{Der2s} representing the Laplacian, 1st and 2nd derivatives, respectively. All these operators are computed if the template parameters are not explicitly specified.\\
