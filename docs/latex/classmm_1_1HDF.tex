\doxysubsection{Detailed Description}
Simplified \href{https://portal.hdfgroup.org/display/support}{\texttt{ HDF5}} I/O utilities. 

This class is a wrapper around the \href{https://portal.hdfgroup.org/pages/viewpage.action?pageId=50073943}{\texttt{ HDF5 C library}}, for simplified reading and writing.

This class represents a \doxylink{classmm_1_1HDF}{HDF} reader and writer\+: it knows which file (open or closed) and which group (open or closed) are the ones that are to be read from/written to. Any calls to {\ttfamily read\texorpdfstring{$\ast$}{*}} or {\ttfamily write\texorpdfstring{$\ast$}{*}} methods write to that group in that file. The file and group can be opened, closed or changed during the lifetime of the object, with user\textquotesingle{}s responsibility being that they are open when writing to/reading from it.

In addition \doxylink{classmm_1_1HDF_a668345f283a7978e2ff29ae7a3ddd532}{atomic()} method can be used to perform {\ttfamily open/read/close} or {\ttfamily open/write/close} cycle in one line.

Raw C identifiers are exposed with \doxylink{classmm_1_1HDF_a281c665e78ad019ef759a458f11fcc14}{get\+File\+ID()} and \doxylink{classmm_1_1HDF_ad626bf0cd3c550b3ee8b6762ffaa1139}{get\+Group\+ID()} methods and can be used to perform any action not directly supported by this class.

All {\ttfamily write} methods create the datasets or attributes required. They also have an {\ttfamily overwrite} parameter which is set to {\ttfamily false} by default. This means that any attempt to write to an existing object will fail an assertion to prevent accidental deletions of existing data. This can be disabled by setting {\ttfamily overwrite = true}, allowing one to overwrite existing data.

All open objects are closed upon destruction.

A group with given name is created and durations (in seconds) between consecutive labels are writen as double attributes named {\ttfamily label1-\/label2}.
